# Astro Wiki Rebuild Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the VitePress runtime with an Astro foundation that can later render `docs/` as an Obsidian-style campus wiki.

**Architecture:** Start with a minimal Astro app in the repository root, remove VitePress-specific runtime files and scripts, then verify the new app builds. Keep all Markdown content under `docs/` intact for later content-index work.

**Tech Stack:** Astro, Bun, TypeScript, Markdown, existing `docs/` content vault.

---

## File Structure

- Remove: `docs/.vitepress/`
  - Old VitePress config, theme, plugins, and components. Current content under `docs/` remains.
- Modify: `package.json`
  - Replace VitePress scripts with Astro scripts.
  - Replace VitePress-only dependencies with Astro baseline dependencies.
- Modify: `bun.lock`
  - Regenerated by `bun install`.
- Create: `astro.config.mjs`
  - Astro site configuration.
- Create: `src/pages/index.astro`
  - Minimal home page proving Astro is initialized.
- Create: `src/styles/global.css`
  - Initial global styling.
- Create: `src/env.d.ts`
  - Astro type references.

## Task 1: Remove VitePress Runtime Surface

**Files:**
- Delete: `docs/.vitepress/`
- Modify: `package.json`
